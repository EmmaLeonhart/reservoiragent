# arXiv source bundle

The arXiv-ready LaTeX source for the **Reservoir Attention Network (RAN)**
paper. The bundle is built by the
[`paper-pdf.yml`](https://github.com/EmmaLeonhart/reservoiragent/actions/workflows/paper-pdf.yml)
workflow and uploaded as the `reservoir-arxiv-source.tar.gz` artifact (alongside
the named + anonymized PDFs).

## How to submit

1. Download `reservoir-arxiv-source.tar.gz` from the latest **Paper PDF**
   workflow run's artifacts.
2. On arXiv's submission form, choose the "upload a single `.tar.gz` / `.zip`"
   option and upload that file.
3. On the **Select Compiler** screen, keep the auto-selection: compiler
   `pdflatex`, TeX Live 2025, top-level `paper.tex`. All files
   (`paper.tex`, `paper.tex.body`, `neurips_2026.sty`, and the `figures/`
   directory) should be marked **Keep**.
4. arXiv runs `pdflatex` (not `pandoc` — pandoc was already run when the bundle
   was built) and produces the public PDF.
5. In the **Comments** metadata field, paste the AI-disclosure one-liner:
   *AI language models were used for literature exploration and drafting
   assistance; experiments, results, and final text are the author's own. Full
   disclosure in the AI-use statement.*

## What's inside, and how it builds

The build chain is:

```text
FINDINGS.md  ─preprocess+pandoc→  paper.tex.body  ─\input→  paper.tex  ─pdflatex→  paper.pdf
                                                                ↑
                                              neurips_2026.sty  +  figures/
```

`FINDINGS.md` at the repo root is the single canonical source — the GitHub-Pages
report and the clawRxiv submission read the same file. arXiv's side only runs the
second half (`pdflatex paper.tex` against the files in the tarball), so the bundle
ships pandoc's output pre-generated:

- **`paper.tex`** — the wrapper: documentclass, packages, hyperref config,
  Unicode → glyph mapping, the `graphicx` max-width clamp for figures, the
  pandoc-longtable + hyperref `none`-counter shim, title and author block,
  `\input{paper.tex.body}`.
- **`paper.tex.body`** — the entire paper body as LaTeX, produced by pandoc from
  a preprocessed copy of `FINDINGS.md` on every CI run. The preprocessing strips
  the H1 title (rendered by `\maketitle`), tags Abstract / References /
  Limitations as unnumbered, and rewrites `docs/<fig>` image paths to the flat
  `figures/<fig>` paths shipped in the tarball.
- **`neurips_2026.sty`** — the official NeurIPS 2026 style file, loaded in
  `preprint` mode (named author block, no line numbers).
- **`figures/`** — the figures `\includegraphics`'d by the body: PNGs copied
  as-is, and the SVG diagrams converted to PDF (pdflatex cannot read SVG) so they
  stay vector.

## Two builds from one source

The `\ifanon` switch in `paper.tex` produces both the named preprint and the
anonymized submission from the same source:

```text
pdflatex paper.tex                                     -> named (preprint, no line numbers)
pdflatex -usepretex='\newif\ifanon\anontrue' paper.tex -> anonymized (Anonymous Authors, line numbers)
```

The arXiv preprint uses the named (preprint) build.

## Reproducibility

The reproduction surface referenced in the paper is the `reproduce-report`
SKILL.md and the scripts under `scripts/` / `replication/` in the repository.
You can rely on the in-paper GitHub URL, or attach the relevant scripts as arXiv
ancillary files alongside this source bundle.

This page and the artifact are regenerated by the
[Paper PDF workflow](https://github.com/EmmaLeonhart/reservoiragent/actions/workflows/paper-pdf.yml)
on every change to `FINDINGS.md` or `paper/`.
